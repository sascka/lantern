# Проверка границ Lantern v0.1

Дата проверки реализации: 18 июля 2026 года.

Это внутренний отчёт по критериям MVP, а не аудит безопасности. Он отделяет
автоматически проверенное поведение от ручной демонстрации и внешних проверок.

## Критерии MVP

| N | Состояние | Основание |
| --- | --- | --- |
| 1 | Проверено на loopback | Интеграционный тест запускает отдельные CLI-процессы и три постоянных узла. Три физических компьютера остаются ручной проверкой. |
| 2 | Проверено с ограничением | Два процесса выполняют подписанные QR, одинаковый SAS и два подтверждения. Тест сравнивает SAS, но внимательность человека доказать не может. |
| 3 | Проверено | Alice создаёт сообщение до запуска сетевой встречи с Bob. |
| 4 | Проверено | Relay принимает только ограниченный канонический Envelope и хранит его в открытой очереди. |
| 5 | Проверено | Встреча Alice с Relay полностью завершается до встречи Relay с Bob. |
| 6 | Проверено | Bob коммитит проверенный decrypt и показывает новое сообщение один раз. |
| 7 | Проверено в границах приложения | У Relay нет secret profile и API decrypt. Тест ищет тестовые пароли и plaintext в выводе и файлах Relay. Метаданные Envelope остаются видимыми. |
| 8 | Проверено | Негативные тесты изменяют защищённые поля и ciphertext, после чего decrypt отклоняется без commit ratchet. |
| 9 | Проверено | Tombstone и история блокируют replay и второй показ после перезапуска. |
| 10 | Проверено | Размер, версия, TTL и hop limit проверяются строгими декодерами, очередью и тестами границ. |
| 11 | Проверено | Очередь ограничена числом объектов и байтами, FIFO-вытеснение и лимиты SQLite сохраняются после перезапуска. |
| 12 | Проверено в модели MVP | Есть тесты обрыва, повторного соединения, дубликатов, перестановки и транзакционного восстановления. Полное отключение питания не моделировалось. |
| 13 | Проверено | Полный набор тестов прошёл на чистом коммите `31bae2a49153833371138d34903a699885a18f03`. |
| 14 | Проверено | Скрипт успешно выполнил `cargo fmt`, `cargo clippy`, release-сборку, `cargo test`, ruff и pytest. |
| 15 | Проверено в воспроизводимой среде | `DEMO.md` содержит ручную последовательность, а скрипт завершился фактическим результатом `passed` на указанном коммите. |

## Автоматическая сквозная проверка

Тест `apps/lantern-cli/tests/three_process_demo.rs` использует настоящий CLI,
Olm payload, TCP loopback, постоянные очереди и SQLCipher-профили. В нём нет
прямого соединения Alice с Bob. Relay перезапускается логически между двумя
отдельными командами и не получает парольную фразу или secret profile.

Автоматический тест не заменяет ручную проверку на трёх машинах. Он не проверяет
настройки файрвола, разные сетевые карты, повреждение файловой системы или
потерю питания.

## Предварительный результат

18 июля 2026 года на текущем незакоммиченном дереве фактически выполнены:

```text
cargo fmt --all --check                                      passed
cargo clippy --workspace --all-targets --all-features ...    passed
cargo test --workspace --all-targets --all-features ...      278 passed
python -m ruff format --check lantern_sim tests               passed
python -m ruff check lantern_sim tests                        passed
python -m pytest                                              141 passed
```

В Rust-набор входит сквозной тест отдельных CLI-процессов, который занял 18.65
секунды в этом запуске. Это честный предварительный результат, но он не закрывает
критерии 13-15: рабочее дерево ещё не было чистым, а результат не привязан к
хешу коммита.

Первая release-сборка обнаружила ошибочные TLS relocation в объектном файле
SQLCipher, собранном GCC 16.1.1. После фиксации Clang как C-компилятора команда
`cargo build -p lantern-cli --release --locked` прошла за 30.59 секунды. После
этой настройки строгий clippy и все 278 Rust-тестов прошли повторно. Release CLI
также запустился и вывел справку experimental preview.

Рабочее дерево перед запуском было чистым, иначе скрипт завершился бы до первой

физических компьютерах и независимый аудит по-прежнему не выполнены.

## Что эта проверка не гарантирует

- независимого аудита криптографии и всей реализации не было;
- fuzzing всех сетевых и файловых декодеров не завершён;
- устойчивость к вредоносному процессу с правами локального пользователя не
  заявляется;
- Relay может удалить, задержать или анализировать размеры и время трафика;
- неверные часы могут преждевременно удалить сообщение;
- защита от повторов ограничена размером локального хранилища;
- Lantern нельзя использовать для реальной опасной переписки.
