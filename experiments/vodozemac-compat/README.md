# Проверка vodozemac 0.10.0

Это отдельный исследовательский crate. Он не входит в основной Cargo workspace
Lantern и не подключён к `lantern-node`, хранилищу или транспорту.

Здесь проверяется только стабильный Olm v1 через
`SessionConfig::version_1()`. Default features отключены, поэтому
экспериментальный Olm v2 недоступен при сборке.

## Что проверяется

- создание асинхронной сессии через one-time key;
- ответ после первого pre-key сообщения;
- изменение каждого байта закодированного сообщения;
- сохранение состояния после неудачной расшифровки;
- повтор уже принятого сообщения;
- обратная перестановка 32 сообщений;
- фактический предел в 40 сохранённых пропущенных ключей;
- сохранение и восстановление Account и Session через зашифрованные pickle;
- отказ при неправильном ключе pickle;
- содержимое `Debug` у объектов на границе API.

Тестовый ключ pickle не является секретом и никогда не должен использоваться
в настоящем приложении. Реальные ключи создаются через системный генератор
случайных чисел. Фиксировать seed для криптографических ключей нельзя.

## Запуск

Из корня репозитория:

```bash
cd experiments/vodozemac-compat
cargo fmt --all -- --check
cargo clippy --all-targets --locked -- -D warnings
cargo test --all-targets --locked
```

Эксперимент не доказывает безопасность Olm или Lantern. Его задача - проверить
совместимость API и явно записать ограничения до архитектурного решения.

Фактические результаты находятся в [RESULTS.md](RESULTS.md).
