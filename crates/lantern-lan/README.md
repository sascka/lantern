# lantern-lan

Первый сетевой пакет Этапа 3. Он соединяет два Linux-процесса по вручную
заданному TCP-адресу и проверяет, что обе стороны поддерживают LAN-протокол
версии 1.

Crate пока не передаёт `Envelope`, списки контейнеров или сообщения. Соединение
закрывается сразу после успешного `Hello`.

## Границы

- принимаются только loopback, IPv4 private, IPv4 link-local, IPv6 unique-local
  и IPv6 link-local адреса;
- DNS, wildcard, multicast и публичные адреса не принимаются;
- `Hello` всегда занимает 8 байт;
- на чтение и запись `Hello` действует таймаут 5 секунд;
- неизвестная версия и любое ненулевое служебное поле отклоняются;
- ошибки и `Debug` не содержат IP-адрес, порт или текст ошибки ОС;
- входящий `Hello` читается в фиксированный массив без выделения памяти.

Ограничение адресов уменьшает вероятность случайного подключения через
интернет, но не проверяет физическое расположение узла. Маршрутизацию определяет
операционная система.

## Ручная проверка на одном компьютере

В первом терминале:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo run -p lantern-lan --bin lantern-lan-probe --locked -- \
  listen 127.0.0.1:38383
```

Во втором терминале:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo run -p lantern-lan --bin lantern-lan-probe --locked -- \
  connect 127.0.0.1:38383
```

Оба процесса должны вывести:

```text
Lantern LAN version 1 negotiated
```

Для двух устройств вместо loopback нужно указать частный адрес интерфейса
слушающего устройства. Узнать его на Arch Linux можно командой:

```bash
ip -brief address
```

Probe не является пользовательским интерфейсом или фоновым сервисом. Он нужен
только для проверки текущего сетевого шага.

## Автоматические проверки

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo fmt --all -- --check
cargo clippy -p lantern-lan --all-targets --locked -- -D warnings
cargo test -p lantern-lan --all-targets --locked
```

Тесты используют loopback и проверяют успешный обмен, фрагментированный
`Hello`, неизвестную версию, зависший узел, ограничения адресов и безопасный
вывод ошибок.

## Ограничения

- TCP-соединение не аутентифицировано и не зашифровано;
- IP-адрес виден операционной системе и наблюдателю сети, хотя Lantern его не
  записывает;
- один удалённый узел может занимать listener до 5 секунд;
- нет общего rate limit по IP;
- нет wire framing для `Envelope`;
- нет повторного подключения и обнаружения соседей;
- проект не проходил независимый аудит.
