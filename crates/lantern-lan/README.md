# lantern-lan

Первые два сетевых пакета Этапа 3. Crate соединяет два Linux-процесса по
вручную заданному TCP-адресу, согласовывает LAN-протокол версии 1 и передаёт
ограниченные непрозрачные кадры через общий `FrameTransport`.

Crate пока не передаёт `Envelope`, списки контейнеров или сообщения. Временный
probe использует только заранее известные синтетические байты.

## Границы

- принимаются только loopback, IPv4 private, IPv4 link-local, IPv6 unique-local
  и IPv6 link-local адреса;
- DNS, wildcard, multicast и публичные адреса не принимаются;
- `Hello` всегда занимает 8 байт;
- один deadline 5 секунд покрывает отправку и чтение полного `Hello`;
- неизвестная версия и любое ненулевое служебное поле отклоняются;
- каждый кадр имеет 4-байтный big-endian prefix длины;
- тело занимает от 1 байта до 64 KiB;
- prefix проверяется до тела без выделения памяти по недоверенной длине;
- один общий deadline 5 секунд покрывает весь кадр;
- частичный кадр никогда не возвращается вызывающему коду;
- после framing-ошибки соединение больше не используется;
- `BoundedSession` ограничивает кадры и байты в обоих направлениях;
- ошибки и `Debug` не содержат IP-адрес, порт или текст ошибки ОС;
- входящий `Hello` читается в фиксированный массив без выделения памяти.

Ограничение адресов уменьшает вероятность случайного подключения через
интернет, но не проверяет физическое расположение узла. Маршрутизацию определяет
операционная система.

## Ручная проверка на одном компьютере

В первом терминале:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo run -p lantern-lan --bin lantern-lan-probe --locked -- \
  listen 127.0.0.1:38383
```

Во втором терминале:

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo run -p lantern-lan --bin lantern-lan-probe --locked -- \
  connect 127.0.0.1:38383
```

Оба процесса должны вывести:

```text
Lantern LAN version 1 and opaque frame verified
```

Для двух устройств вместо loopback нужно указать частный адрес интерфейса
слушающего устройства. Узнать его на Arch Linux можно командой:

```bash
ip -brief address
```

Probe не является пользовательским интерфейсом или фоновым сервисом. Он нужен
только для проверки текущего сетевого шага.

## Автоматические проверки

```bash
export PATH="$HOME/.cargo/bin:$PATH"
cargo fmt --all -- --check
cargo clippy -p lantern-lan --all-targets --locked -- -D warnings
cargo test -p lantern-lan --all-targets --locked
```

Тесты используют loopback и проверяют handshake, точные wire vectors,
фрагментированные prefix и body, общий deadline, чистое закрытие, частичный
кадр, нулевую и завышенную длину, точный максимум 64 KiB, ограничения адресов
и безопасный вывод ошибок. Property-тест перебирает произвольные prefix длины.

## Ограничения

- TCP-соединение не аутентифицировано и не зашифровано;
- IP-адрес виден операционной системе и наблюдателю сети, хотя Lantern его не
  записывает;
- один удалённый узел может занимать listener до 5 секунд;
- нет общего rate limit по IP;
- framing раскрывает точный размер тела наблюдателю сети;
- служебные кадры синхронизации пока не определены;
- `Envelope` пока не передаётся и не сохраняется через сеть;
- нет повторного подключения и обнаружения соседей;
- проект не проходил независимый аудит.
