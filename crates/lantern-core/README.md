# Lantern Core

Статус: первый рабочий пакет Этапа 2.

`lantern-core` содержит ограниченные доменные типы, которые не зависят от сети,
хранилища и криптографической библиотеки.

Сейчас crate умеет:

- проверять логические поля Envelope версии 1;
- ограничивать TTL, hop limit, priority и размер непрозрачного payload;
- хранить `message_id` и `recipient_hint` как отдельные типы фиксированного
  размера;
- создавать локальную маршрутную запись для источника и Relay;
- ограничивать недоверенные маршрутные счётчики локальными максимумами;
- проверять переходы состояния контейнера;
- скрывать полные идентификаторы, подсказку получателя, payload и точные
  локальные времена из `Debug`.

## Что пока не реализовано

- CBOR-сериализация и декодирование;
- проверка общего сериализованного размера;
- генерация случайного `message_id`;
- создание `recipient_hint`;
- E2EE и работа с настоящим текстом;
- очередь, дедупликация и tombstone;
- постоянное хранилище;
- расчёт TTL во время работы и после перезапуска;
- интерфейс транспорта и сеть.

Массивы в тестах являются очевидными тестовыми значениями. Они не являются
ключами, шифротекстом или защищённой пользовательской перепиской.

## Граница недоверенного ввода

`Envelope::try_from_fields` принимает уже разделённые поля. Он проверяет их
значения, но вызывающий код уже создал `Vec<u8>` для payload. Поэтому этот API
не является безопасным CBOR-декодером и не доказывает, что большой вход будет
отклонён до выделения памяти.

Будущий декодер обязан сначала ограничить весь вход значением 64 KiB, затем
проверить объявленные длины CBOR и только после этого создавать эти типы.

## Время

`first_seen_at` и `local_deadline` пока представлены локальными секундами в
одной шкале времени. При создании записи проверяется переполнение сложения.

Точная связь монотонных часов, настенных часов и сохранённого срока после
перезапуска ещё не выбрана. До отдельного решения эти значения нельзя считать
готовой реализацией устойчивого TTL.

## Запуск на Arch Linux

Если `rustup` ещё не установлен:

```bash
sudo pacman -S --needed rustup
```

Затем:

```bash
cd "/home/qual/Desktop/prog/Lantern proj"

export PATH="$HOME/.cargo/bin:$PATH"
rustup set profile minimal
rustup toolchain install 1.97.1 --component clippy,rustfmt

cargo fmt --all --check
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-targets
```

Файл `rust-toolchain.toml` фиксирует Rust 1.97.1. У crate пока нет сторонних
зависимостей.

## Ограничения безопасности

Этот пакет не обеспечивает конфиденциальность или целостность сообщения.
`protected_payload` является только непрозрачным массивом байтов. Relay с
доступом к памяти текущей реализации увидит весь Envelope и локальную
маршрутную запись.

Кража базы пока не рассматривается в коде, потому что базы нет. Изменение и
повтор байтов пока обнаруживаются только будущими CBOR-, дедупликационным и
криптографическим компонентами.

Жёсткие диапазоны уменьшают риск бесконечных маршрутных значений, но очередь и
ограничение общего потребления памяти ещё не реализованы. В коде нет логов,
сетевых обращений, `unsafe` и секретных данных.
