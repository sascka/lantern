# Lantern CLI

`lantern-cli` - минимальное Linux-приложение для демонстрации Lantern v0.1.
Оно создаёт профиль, добавляет контакт, шифрует короткое сообщение и переносит
его через локальный Relay.

Это experimental preview. Приложение не проходило независимый аудит и не
подходит для реальной опасной или экстренной переписки.

## Сборка на Arch Linux

```bash
sudo pacman -S --needed rustup base-devel clang git

cd "/home/qual/Desktop/prog/Lantern proj"
rustup toolchain install 1.97.1 --profile minimal \
  --component clippy,rustfmt
cargo build -p lantern-cli --release --locked
./target/release/lantern-cli help
```

Версия Rust зафиксирована в `rust-toolchain.toml`, а версии зависимостей - в
`Cargo.lock`. Встроенный SQLCipher собирается через Clang. GCC 16.1.1 на
текущем Arch Linux создаёт некорректные TLS relocation для его C-кода в
release-режиме, поэтому компилятор C зафиксирован в `.cargo/config.toml`.

## Команды

```text
profile-init PROFILE PASSFILE
node-init NODE
contact-invite PROFILE PASSFILE NAME INVITE RESPONSE CONFIRM_IN CONFIRM_OUT
contact-respond PROFILE PASSFILE NAME INVITE RESPONSE CONFIRM_OUT CONFIRM_IN
contacts PROFILE PASSFILE
send PROFILE PASSFILE NODE CONTACT MESSAGEFILE TTL MAX_HOPS
listen NODE BIND_ADDRESS
connect NODE PEER_ADDRESS
receive PROFILE PASSFILE NODE
inbox PROFILE PASSFILE
diagnostics NODE
```

Полный последовательный пример находится в корневом файле
[`DEMO.md`](../../DEMO.md).

## Где хранятся данные

У конечного пользователя два разных каталога:

- `PROFILE` - SQLCipher-база с идентичностью, контактами, сессиями и историей;
- `NODE` - открытая транспортная очередь и ограниченная диагностика.

Relay получает только каталог `NODE`. У него нет профиля и API для открытия
сообщения. При взломе Relay противник увидит зашифрованные контейнеры, размеры,
сроки и часть маршрутных метаданных. Он также сможет удалить, задержать или не
передать сообщение.

Каталоги профиля и узла должны иметь права `0700`. Файлы с парольной фразой,
сообщением и данными контактного обмена - `0600`. Символические ссылки, жёсткие
ссылки и слишком большие файлы отклоняются. Пароль и текст не передаются через
аргументы командной строки или переменные окружения.

## Добавление контакта

Alice запускает `contact-invite`, Bob одновременно запускает
`contact-respond`. Команды обмениваются четырьмя приватными файлами и показывают
одинаковый SAS из трёх чисел. Обе стороны должны сверить числа по независимому
каналу и ввести `MATCH` только при полном совпадении.

Незавершённый обмен нельзя продолжить после остановки процесса. Это ограничение
библиотеки Olm, а не ошибка сохранения: временное состояние SAS намеренно не
сериализуется. После обрыва нужно удалить файлы обмена и начать заново.

Файловый обмен ограничен двумя минутами. В демонстрации процессы используют
общий локальный каталог. На разных компьютерах эти файлы придётся переносить
вручную по доверенному внеполосному каналу.

## Что появляется в терминале

Обычные команды выводят только короткий статус, SAS, имя контакта, отпечаток
или ограниченные диагностические коды. `receive` и `inbox` намеренно показывают
открытый текст пользователю Bob. Их вывод нельзя перенаправлять в общий журнал.

Диагностика не содержит текста, паролей, ключей, полных идентификаторов
сообщений, путей к базам или адресов соседей. Телеметрии и скрытых сетевых
запросов нет.

## Ограничения

- сеть указывается вручную как IP-адрес и порт;
- `listen` и `connect` выполняют одну встречу и завершаются;
- нет обнаружения соседей, фонового режима, GUI и восстановления ключей;
- неверное время устройства может преждевременно удалить очередь;
- потерянные и переставленные пакеты требуют новой встречи;
- локальный вредоносный процесс с правами пользователя не входит в модель
  защиты CLI;
- автоматические тесты не доказывают, что человек внимательно сверил SAS.
